Requisitos Técnicos para o Projeto de Desenvolvimento de Aplicativos Móveis

1. Descrição do Projeto

Este projeto consiste no desenvolvimento de um aplicativo móvel completo, desde a concepção até a implementação. O aluno deverá aplicar os conhecimentos adquiridos durante a disciplina para criar uma solução que resolva um problema real, seguindo as boas práticas de desenvolvimento mobile.

Tema do Projeto:

O aluno poderá escolher entre as seguintes opções:

  • Aplicativo de gerenciamento de tarefas pessoais
  • Aplicativo de comércio eletrônico
  • Aplicativo de notícias/informações
  • Rede social específica para um nicho
  • Ferramenta de produtividade para estudantes
  • Aplicativo para área de saúde/bem-estar
  • Tema livre (mediante aprovação do professor)

2. Requisitos Técnicos

2.1 Requisitos Fundamentais

  • O aplicativo deve ser desenvolvido para plataformas iOS e/ou Android
  • Implementação de pelo menos 5 telas diferentes com navegação entre elas
  • Persistência de dados (local e/ou remota)
  • Design responsivo adaptável a diferentes tamanhos de tela

2.2 Requisitos Funcionais

  • Sistema de autenticação de usuários (login/cadastro)
  • CRUD completo (Create, Read, Update, Delete) de pelo menos uma entidade
  • Integração com pelo menos uma API externa
  • Implementação de pelo menos um recurso específico de dispositivos móveis:
    • Câmera
    • Geolocalização
    • Notificações push
    • Sensores (acelerômetro, giroscópio, etc.)
    • Armazenamento de arquivos
  • Interface de usuário intuitiva e amigável seguindo diretrizes de UI/UX para mobile

2.3 Requisitos Não-Funcionais

  • Desempenho: o aplicativo deve responder em menos de 2 segundos para operações comuns
  • Segurança: implementação de boas práticas para proteção de dados sensíveis
  • Disponibilidade: o aplicativo deve funcionar offline para funcionalidades básicas
  • Tratamento adequado de erros e exceções
  • Documentação completa do código

3. Tecnologias Recomendadas

3.1 Desenvolvimento Frontend

React Native Flutter Swift Kotlin Ionic

3.2 Desenvolvimento Backend

Firebase Node.js MongoDB MySQL SQLite

3.3 Ferramentas Auxiliares

Git/GitHub Figma/Adobe XD Postman Android Studio Xcode VS Code

4. Etapas de Desenvolvimento

Etapa Descrição Entrega
1. Concepção Definição do tema, escopo, e requisitos específicos Documento de proposta e wireframes iniciais
2. Prototipação Desenvolvimento de protótipos de alta fidelidade Protótipos navegáveis
3. Desenvolvimento Implementação do frontend e backend Código fonte versionado no GitHub
4. Testes Validação das funcionalidades e correção de bugs Relatório de testes realizados
5. Apresentação Demonstração do aplicativo funcionando Aplicativo finalizado e documentação completa

5. Critérios de Avaliação

  • Funcionalidade (30%): Implementação correta dos requisitos funcionais
  • Qualidade técnica (25%): Estrutura do código, arquitetura e boas práticas
  • Interface e UX (20%): Design, usabilidade e experiência do usuário
  • Inovação e criatividade (15%): Elementos diferenciais e soluções criativas
  • Apresentação e documentação (10%): Clareza na apresentação e qualidade da documentação

6. Recursos e Materiais de Apoio

Observações Importantes

  • É permitido o uso de bibliotecas e frameworks de código aberto
  • O trabalho pode ser individual ou em grupos de até 4 pessoas (ajustes nos requisitos serão feitos para trabalhos em grupo)
  • Dúvidas e esclarecimentos devem ser encaminhados ao professor via e-mail institucional
