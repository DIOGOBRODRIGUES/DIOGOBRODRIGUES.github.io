Serbank - Requisitos do Projeto e Ficha de Avaliação

Requisitos do Projeto

Descrição Geral

O aplicativo será um protótipo de um banco digital chamado "SmartBank", permitindo que os usuários visualizem saldo, histórico de transações e realizem operações básicas como transferência e pagamento. O foco é utilizar React Native com TypeScript ou JavaScript.

Requisitos Funcionais

  1. Tela de Login
    • Campos: CPF (somente números) e senha (máscara para ocultar caracteres).
    • Botão "Entrar" que valida os dados (ex.: CPF "12345678900" e senha "1234").
    • Mensagem de erro se os dados estiverem incorretos.
    • Redirecionar para a tela inicial após login.
  2. Tela Inicial (Dashboard)
    • Exibir o saldo atual do usuário.
    • Botões: "Transferir", "Pagar" e "Histórico de Transações".
    • Header com o nome "Serbank" e um ícone.
  3. Tela de Transferência
    • Campos: Número da conta e valor (formato monetário).
    • Botão "Confirmar" exibe alerta com os dados.
    • Atualizar o saldo.
  4. Tela de Pagamento
    • Campos: Código de barras (12 dígitos) e valor.
    • Botão "Pagar" exibe alerta e atualiza o saldo.
  5. Tela de Histórico de Transações
    • Lista (FlatList) com transações simuladas (data, tipo, valor).
    • Itens estilizados com cores diferentes para entrada e saída.

Requisitos Não Funcionais

  • Tecnologia: React Native com TypeScript ou JavaScript, usando React Navigation.
  • Estilização: Usar StyleSheets e componentes estilizáveis reutilizáveis.
  • Gerenciamento de Estado: Hooks (useState, useEffect); opcional: Context API.
  • Apresentação: Rodar no emulador ou dispositivo físico.

Escopo Mínimo

  • 4 telas: Login, Dashboard, Transferência e Histórico.
  • Validação básica nos formulários.
  • Navegação e estilização consistentes.

Opcionais (Extras)

  • Animações com Animated API.
  • Logout na tela inicial.
  • AsyncStorage para persistência de dados.
