📱 O que são Elementos Nativos?

Definição

Elementos nativos são componentes de interface do usuário que são implementados usando o código nativo de cada plataforma (iOS ou Android).

🎨 São os blocos de construção visuais que o sistema operacional fornece.

Exemplos de Elementos Nativos

🍎 iOS (UIKit)

  • UIView - Container básico
  • UILabel - Texto estático
  • UIButton - Botão
  • UITextField - Campo de texto
  • UIImageView - Imagem
  • UIScrollView - Área rolável
  • UITableView - Lista

🤖 Android (Views)

  • View - Container básico
  • TextView - Texto estático
  • Button - Botão
  • EditText - Campo de texto
  • ImageView - Imagem
  • ScrollView - Área rolável
  • ListView/RecyclerView - Lista

🔑 Características Importantes

  • ⚡ Performance nativa: Renderizados diretamente pelo SO
  • 🎨 Look & Feel nativo: Aparência natural de cada plataforma
  • ♿ Acessibilidade: Suporte a recursos nativos
  • 🔧 APIs nativas: Acesso total aos recursos do dispositivo
  • 🎭 Animações nativas: Transições suaves

🖼️ Views no React Native

<View> é o componente fundamental de construção de UI no React Native.

É um container que suporta layout, estilo, gestos e acessibilidade.

import { View } from 'react-native';

<View style={{ 
  flex: 1, 
  backgroundColor: '#fff',
  padding: 20 
}}>
  {/* Outros componentes aqui */}
</View>

📊 View = Container Universal

🌐 Web (React)

<div style={{ padding: 10 }}>
  <p>Texto</p>
</div>

No navegador = elemento <div>

📱 Mobile (React Native)

<View style={{ padding: 10 }}>
  <Text>Texto</Text>
</View>

iOS = UIView
Android = android.view.View

🎯 Funções de uma View

  • 📦 Layout: Organiza elementos filhos usando Flexbox
  • 🎨 Estilo: Cores, bordas, sombras, etc.
  • 👆 Gestos: Toque, arraste, pinça, etc.
  • ♿ Acessibilidade: Labels e hints para leitores de tela
  • 👁️ Visibilidade: Controla exibição e opacidade

📝 Exemplo Prático

import { View, Text, StyleSheet } from 'react-native';

  }

🌉 A Ponte (Bridge) do React Native

O que é a Bridge?

A Bridge é uma camada de comunicação que conecta o código JavaScript com o código nativo de cada plataforma.

🔄 Permite que JavaScript e código nativo conversem usando mensagens assíncronas.


📐 Arquitetura em Camadas

JavaScript Thread
Seu código React Native / Lógica de negócio / Estado do app
⬇️
Bridge (Ponte)
Comunicação assíncrona via JSON / Serialização de mensagens
⬇️
Native Thread
UI nativa / Módulos nativos / APIs do sistema

🔄 Fluxo de Comunicação

// 1. Código JavaScript
      import { Button } from 'react-native';
      <Button title="Clique" onPress={() => console.log('Clicado')} />
⬇️
// 2. Bridge serializa para JSON
      {
        "type": "createView",
        "component": "RCTButton",
        "props": { "title": "Clique" }
      }
⬇️
// 3. Código nativo iOS - Reconstrução (Desserialização)
      let button = UIButton(type: .system)
      button.setTitle("Clique", for: .normal)

⚠️ Limitações da Bridge Tradicional

🔴 Gargalo (Bottleneck): Passar muitos dados via JSON pela Bridge causa lag

  • Imagens pesadas
  • Atualizações de scroll muito rápidas
  • Animações complexas

⏱️ Custo de Conversão: Serializar para JSON e desserializar consome processamento

⏸️ Assincronia Forçada: Tudo é assíncrono via Bridge, dificultando respostas imediatas

Problema: Animações sincronizadas com gestos ficam defasadas


🎯 Solução: JSI (JavaScript Interface)

JSI elimina a necessidade de serializar tudo para JSON

  • ✅ JavaScript mantém referência direta a objetos nativos
  • ✅ Comunicação síncrona quando necessário
  • ✅ Performance muito melhor para operações críticas
  • ✅ Base da nova arquitetura (Fabric + TurboModules)


⚡ Nova Arquitetura (Fabric & TurboModules)

🚀 Fabric: Nova arquitetura de renderização

  • ✅ Comunicação síncrona entre JavaScript e nativo
  • ✅ Melhor performance e menor latência
  • ✅ Suporte a React Concurrent Mode

⚙️ TurboModules: Nova arquitetura de módulos nativos

  • ✅ Carregamento lazy de módulos
  • ✅ Interoperabilidade com JSI (JavaScript Interface)
  • ✅ Eliminação da bridge para operações críticas

🔄 Ciclo de Renderização

1️⃣ Código JavaScript

function App() {
  const [count, setCount] = useState(0);
  
  return (
    <View>
      <Text>Contador: {count}</Text>
      <Button 
        title="Incrementar" 
        onPress={() => setCount(count + 1)} 
      />
    </View>
  );
}


2️⃣ Virtual DOM

React Native cria uma árvore de elementos virtual (semelhante ao Virtual DOM do React)

{
  type: 'View',
  props: { style: {...} },
  children: [
    {
      type: 'Text',
      props: { children: 'Contador: 0' }
    },
    {
      type: 'Button',
      props: { 
        title: 'Incrementar',
        onPress: [Function]
      }
    }
  ]
}

3️⃣ Shadow Tree

Calcula o layout usando Yoga (implementação Flexbox em C++)

  • 📏 Calcula posições e dimensões
  • 📐 Aplica Flexbox
  • 🎨 Resolve estilos

4️⃣ Native Views

Envia instruções para o thread nativo criar/atualizar os componentes visuais

💻 Exemplo Prático Completo

App Todo List Simples

import React, { useState } from 'react';
import { 
  View, Text, TextInput, Button, FlatList, 
  StyleSheet, TouchableOpacity 
} from 'react-native';

function TodoApp() {
  const [task, setTask] = useState('');
  const [todos, setTodos] = useState([]);

  const addTodo = () => {
    if (task.trim()) {
      setTodos([...todos, { id: Date.now(), text: task }]);
      setTask('');
    }
  };

  const removeTodo = (id) => {
    setTodos(todos.filter(todo => todo.id !== id));
  };

  return (
    <View style={styles.container}>
      <Text style={styles.title}>Minhas Tarefas</Text>
      
      <View style={styles.inputContainer}>
        <TextInput
          style={styles.input}
          value={task}
          onChangeText={setTask}
          placeholder="Nova tarefa..."
        />
        <Button title="Adicionar" onPress={addTodo} />
      </View>

      <FlatList
        data={todos}
        keyExtractor={item => item.id.toString()}
        renderItem={({ item }) => (
          <TouchableOpacity 
            style={styles.todoItem}
            onPress={() => removeTodo(item.id)}
          >
            <Text>{item.text}</Text>
            <Text style={styles.deleteText}>✕</Text>
          </TouchableOpacity>
        )}
      />
    </View>
  );
}

📱 O que Acontece nos Bastidores?

  1. View → UIView (iOS) / android.view.View (Android)
  2. Text → UILabel (iOS) / TextView (Android)
  3. TextInput → UITextField (iOS) / EditText (Android)
  4. Button → UIButton (iOS) / Button (Android)
  5. FlatList → UITableView (iOS) / RecyclerView (Android)
  6. TouchableOpacity → View nativa com gesture recognizers
