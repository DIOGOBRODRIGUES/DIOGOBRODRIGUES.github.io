🟢 O que é Node.js?

🟢 O que é Node.js?

"Runtime JavaScript baseado no motor V8 do Chrome que permite executar JavaScript no servidor"

Características:

  • ⚡ Assíncrono e orientado a eventos
  • 🔀 Single-threaded com event loop
  • 📦 Ecossistema rico (npm)
  • 🚀 Alta performance para I/O

Node.js vs Navegador

🌐 Navegador

  • Tem DOM e BOM
  • window object
  • Restrições de segurança
  • Client-side only

⚙️ Node.js

  • Sem DOM/BOM
  • global object
  • Acesso ao filesystem
  • Server-side

📘 Por que TypeScript?

JavaScript vs TypeScript

JavaScript


function soma(a, b) {
  return a + b;
}

soma(5, 3);      // 8
soma(5, "3");    // "53" 😱
soma(5);         // NaN 😱

TypeScript


function soma(a: number, b: number): number {
  return a + b;
}

soma(5, 3);      // 8
soma(5, "3");    // ❌ Erro!
soma(5);         // ❌ Erro!

✨ Benefícios do TypeScript

  • 🛡️ Type Safety: Detecta erros em tempo de desenvolvimento
  • 🧠 IntelliSense: Autocompletar melhor no VS Code
  • 📚 Self-documenting: Tipos servem como documentação
  • ♻️ Refatoração: Mudanças mais seguras
  • 🔍 Erros em tempo real: Antes de executar
  • 🚀 Escalabilidade: Melhor para projetos grandes

⚡ Express.js

Framework web minimalista e flexível

app.ts

Configuração da aplicação Express


import express from 'express';
import cors from 'cors';
import { router } from './routes.js';

export const app = express();

// Middlewares globais
app.use(express.json());
app.use(cors());

// Rotas
app.use(router);

routes.ts

Definição das rotas


import { Router } from 'express';
import type { Request, Response } from 'express';

const router = Router();

// Rota de teste
router.get('/', (req: Request, res: Response) => {
  res.json({ message: 'API Agro E-commerce 🚀' });
});

export { router };

server.ts

Inicialização do servidor


import { app } from './app.js';

const PORT = process.env.PORT || 3000;

app.listen(PORT, () => {
  console.log(`🚀 Servidor rodando na porta ${PORT}`);
  console.log(`📍 http://localhost:${PORT}`);
});

🔥 Hot Reload com tsx

O que é Hot Reload?

Reinicia automaticamente o servidor quando você salva um arquivo

Ferramentas:

  • tsx: Executa TypeScript diretamente (recomendado)
  • ts-node-dev: Alternativa popular
  • nodemon + ts-node: Combinação clássica

Testando Hot Reload

1. Mantenha o servidor rodando (npm run dev)

2. Edite routes.ts:


router.get('/', (req: Request, res: Response) => {
  res.json({ 
    message: 'API Agro E-commerce 🚀',
    version: '1.0.0'  // Nova propriedade
  });
});

3. Salve o arquivo (Ctrl+S)

4. Recarregue o navegador → mudanças aplicadas!

🔐 Variáveis de Ambiente com dotenv

Por que usar .env?

  • 🔒 Segurança: Não expor credenciais no código
  • 🔄 Flexibilidade: Diferentes configs para dev/prod
  • 📝 Centralização: Todas configs em um lugar
  • 🚫 Gitignore: Nunca fazer commit de segredos

Criar arquivo .env


# Criar .env na raiz do projeto
touch .env
          

# Conteúdo do .env
PORT=3000
NODE_ENV=development
DATABASE_URL="postgresql://user:password@localhost:5432/agrodb"
JWT_SECRET="seu_segredo_super_secreto_aqui"

Usar variáveis no código

Instalar dotenv:


npm install dotenv

Atualizar server.ts:


import 'dotenv/config';
import { app } from './app.js';

const PORT = process.env.PORT || 3000;
const ENV = process.env.NODE_ENV || 'development';

app.listen(PORT, () => {
  console.log(`🚀 Servidor [${ENV}] na porta ${PORT}`);
});

⚠️ Importante!

✅ Fazer commit: .env.example

❌ NUNCA fazer commit: .env


# Criar .env.example (sem valores reais)

echo ".env" >> .gitignore

🐛 Debugging no VS Code

Configurar Debugger

Criar .vscode/launch.json:


{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Debug TypeScript",
      "type": "node",
      "request": "launch",
      "runtimeExecutable": "npm",
      "runtimeArgs": ["run", "dev"],
      "console": "integratedTerminal",
      "skipFiles": ["/**"]
    }
  ]
}

Usando Breakpoints

  1. Clique na margem esquerda do editor (bola vermelha)
  2. Pressione F5 para iniciar o debug
  3. Execute uma requisição que passe pelo breakpoint
  4. Inspecione variáveis e execute linha por linha

router.get('/debug', (req: Request, res: Response) => {
  const data = { message: 'teste' }; // ⭕ Breakpoint aqui
  console.log(data);
  res.json(data);
});
