📥 Request Object (req)

📥 Request Object (req)

Contém todas as informações da requisição HTTP


// Estrutura básica de uma rota
router.get('/exemplo', (req: Request, res: Response) => {
  
  req.method        // 'GET'
  req.url           // '/exemplo?search=test'
  req.headers       // { 'content-type': 'application/json', ... }
  req.params        // Parâmetros de rota { id: '123' }
  req.query         // Query strings { search: 'test' }
  req.body          // Corpo da requisição (POST/PUT)
  req.ip            // IP do cliente
  req.protocol      // 'http' ou 'https'
  req.hostname      // 'example.com'
  req.path          // '/exemplo'
  req.cookies       // Cookies (com middleware)
});

📤 Response Object (res)

📤 Response Object (res)

Métodos para enviar respostas ao cliente


router.get('/response-examples', (req, res) => {
  // JSON (mais comum em APIs)
  res.json({ message: 'Hello' });
  
  // Texto simples
  res.send('Hello World');
  
  // Status + JSON
  res.status(201).json({ created: true });
  
  // Redirect
  res.redirect('/nova-rota');
  
  // Download de arquivo
  res.download('/path/to/file.pdf');
  
  // Definir header customizado
  res.set('X-Custom-Header', 'valor').json({ ok: true });
});

Métodos Principais do Response

res.json() - Envia resposta JSON 

res.json({ user: 'João', age: 25 });
res.status() - Define código de status 

res.status(404).json({ error: 'Not found' });
res.send() - Envia resposta (vários tipos) 

res.send('Hello'); // ou res.send({ json: true });

🔗 Route Parameters (Params)

Parâmetros dinâmicos na URL

Sintaxe e Uso


// Definir rota com parâmetro
// :id é o parâmetro
router.get('/users/:id', (req, res) => {
  
  const userId = req.params.id;
  
  res.json({
    message: `Buscando usuário ${userId}`
  });
});

✅ Requisição: GET /users/123

✅ req.params.id = "123"

Múltiplos Parâmetros


// Rota com múltiplos params
router.get('/products/:category/:id', (req, res) => {
  
  const { category, id } = req.params;
  
  res.json({
    message: `Produto ${id} da categoria ${category}`,
    category,
    id
  });
});

✅ Requisição: GET /products/electronics/456

✅ category = "electronics", id = "456"

⚠️ Importante sobre Params

  • Params sempre são strings
  • Converter quando necessário: Number(id)
  • Validar se existe: if (!id) return res.status(400)...
  • Usar nomes descritivos: :userId melhor que :id

🔍 Query Strings

Parâmetros opcionais após o ?

🔍 Query Strings


router.get('/products', (req, res) => {
  
  // Acessar query params
  const { search, category, minPrice, maxPrice } = req.query;
  
  res.json({
    filters: {
      search,
      category,
      minPrice,
      maxPrice
    },
    message: 'Filtrando produtos...'
  });
});

✅ Requisição: GET /products?search=alface&category=verduras&minPrice=2&maxPrice=10

✅ req.query = { search: 'alface', category: 'verduras', minPrice: '2', maxPrice: '10' }

Query String vs Params

Route Params

  • Obrigatórios
  • Parte da estrutura da URL
  • /users/:id
  • Identificam recursos

Query Strings

  • Opcionais
  • Após o ?
  • ?page=2&limit=10
  • Filtros, ordenação, paginação

📦 Request Body

Dados enviados no corpo da requisição (POST/PUT/PATCH)

Necessário para parsear JSON no body

import express from 'express';

app.use(express.json()); // 👈 Middleware global

router.post('/users', (req, res) => {
  const { name, email, password } = req.body;
  // Agora podemos acessar req.body!

Exemplo: Criar Usuário


router.post('/users', (req, res) => {
  const { name, email, password } = req.body;
  
  // Validação básica
  if (!name || !email || !password) {
    return res.status(400).json({
      error: 'Campos obrigatórios: name, email, password'
    });
  }
  
  // Criar usuário (simulado)
  const newUser = {
    id: Math.random().toString(36).substr(2, 9),
    name,
    email,
    createdAt: new Date()
  };
  
  // Retornar 201 Created
  return res.status(201).json({
    message: 'Usuário criado com sucesso',
    user: newUser
  });
});

Atualização parcial

🔧 Middlewares

O coração do Express

O que são Middlewares?

"Funções que têm acesso aos objetos request, response e à próxima função middleware no ciclo request-response"

Podem:

  • Executar qualquer código
  • Modificar req e res
  • Finalizar o ciclo request-response
  • Chamar o próximo middleware com next()

Anatomia de um Middleware


function meuMiddleware(req: Request, res: Response, next: NextFunction) {
  // 1. Executar código antes da rota
  console.log('Middleware executado!');
  console.log('Método:', req.method);
  console.log('URL:', req.url);
  
  // 2. Modificar request/response (opcional)
  req.customData = 'algum valor';
  
  // 3. Passar para o próximo middleware/rota
  next();
  
  // OU finalizar a requisição
  // res.json({ error: 'Bloqueado' });
}

Tipos de Middlewares

1. Application-level 
app.use(express.json()); // Todas as rotas
2. Router-level 
router.use(authMiddleware); // Rotas do router
3. Route-specific 
router.get('/protected', auth, (req, res) => {});
4. Error-handling 
app.use((err, req, res, next) => { /* ... */ });

Middleware: Logger


// middlewares/logger.ts
export function logger(req: Request, res: Response, next: NextFunction) {
  const timestamp = new Date().toISOString();
  console.log(`[${timestamp}] ${req.method} ${req.url}`);
  next();
}

// app.ts
import { logger } from './middlewares/logger';
app.use(logger); // Todas as requisições passam por aqui

Middleware: Validação


// middlewares/validateUser.ts
export function validateUser(req: Request, res: Response, next: NextFunction) {
  const { name, email, password } = req.body;
  
  if (!name || !email || !password) {
    return res.status(400).json({
      error: 'Campos obrigatórios: name, email, password'
    });
  }
  
  // Se passou, continua
  next();
}

// routes.ts
router.post('/users', validateUser, (req, res) => {
  // Se chegou aqui, dados estão válidos!
  const { name, email, password } = req.body;
  // ... criar usuário
});

Múltiplos Middlewares


function middleware1(req, res, next) {
  console.log('Middleware 1');
  next();
}

function middleware2(req, res, next) {
  console.log('Middleware 2');
  next();
}

router.get('/multi', 
  middleware1,
  middleware2,
  (req, res) => {
    console.log('Handler final');
    res.json({ ok: true });
  }
);

Ordem de execução: middleware1 → middleware2 → handler

⚠️ Tratamento de Erros

Middleware de Erro

Tem 4 parâmetros (incluindo err)


// middlewares/errorHandler.ts
import type { Request, Response, NextFunction } from 'express';

export function errorHandler(
  err: Error,
  req: Request,
  res: Response,
  next: NextFunction
) {
  console.error('Erro:', err.message);
  console.error('Stack:', err.stack);
  
  res.status(500).json({
    error: 'Erro interno do servidor',
    message: err.message
  });
}

// app.ts (deve ser o ÚLTIMO middleware)
app.use(errorHandler);

Try-Catch em Rotas


router.get('/users/:id', async (req, res, next) => {
  try {
    
    const user = await db.user.findUnique({
      where: { id: req.params.id }
    });
    
    if (!user) {
      return res.status(404).json({ error: 'Usuário não encontrado' });
    }
    
    res.json(user);
    
  } catch (error) {
    // Passa o erro para o middleware de erro
    next(error);
  }
});

Custom Error Class


// errors/AppError.ts
export class AppError extends Error {
  statusCode: number;
  
  constructor(message: string, statusCode = 400) {
    super(message);
    this.statusCode = statusCode;
  }
}

// Uso nas rotas
if (!user) {
  throw new AppError('Usuário não encontrado', 404);
}

// Middleware de erro atualizado
export function errorHandler(err, req, res, next) {
  if (err instanceof AppError) {
    return res.status(err.statusCode).json({ error: err.message });
  }
  
  // Erro genérico
  res.status(500).json({ error: 'Erro interno' });
}

Rota 404 (Not Found)


// app.ts
// Depois de todas as rotas válidas
app.use((req, res) => {
  res.status(404).json({
    error: 'Rota não encontrada',
    path: req.url
  });
});

// Middleware de erro por último
app.use(errorHandler);

📁 Organizando Rotas

Estrutura Recomendada


src/
├── app.ts                    # Configuração do Express
├── server.ts                 # Inicialização do servidor
├── routes/
│   ├── index.ts             # Agrupa todas as rotas
│   ├── user.routes.ts       # Rotas de usuários
│   ├── product.routes.ts    # Rotas de produtos
│   └── order.routes.ts      # Rotas de pedidos
├── middlewares/
│   ├── auth.ts              # Autenticação
│   ├── errorHandler.ts      # Tratamento de erros
│   └── validator.ts         # Validações
└── controllers/
    ├── UserController.ts
    ├── ProductController.ts
    └── OrderController.ts

routes/user.routes.ts


import { Router } from 'express';
import { UserController } from '../controllers/UserController';

const userRouter = Router();

userRouter.post('/', userController.create);
userRouter.get('/:id', userController.show);

userRouter.delete('/:id', userController.delete);
