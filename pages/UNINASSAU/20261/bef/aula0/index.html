💻 Front-End vs Back-End

Front-End

  • Interface do usuário
  • Executa no navegador
  • HTML, CSS, JavaScript
  • React, Vue, Angular
  • Design e experiência

Back-End

  • Lógica de negócio
  • Executa no servidor
  • Node.js, Python, Java
  • Express, Django, Spring
  • Dados e segurança

🖥️ O que é Desenvolvimento Back-End?

É a parte "invisível" de uma aplicação web que lida com:

  • Lógica de Negócio: Regras e processamentos
  • Banco de Dados: Armazenamento e recuperação
  • APIs: Comunicação entre sistemas
  • Autenticação: Segurança e controle de acesso
  • Servidor: Hospedagem e processamento

🔄 Fluxo de uma Requisição HTTP

  1. Cliente: Usuário acessa www.exemplo.com/produtos
  2. Request: Navegador envia GET /produtos
  3. Servidor: Recebe e processa a requisição
  4. Banco de Dados: Consulta produtos
  5. Lógica: Formata os dados
  6. Response: Retorna JSON com produtos
  7. Cliente: Renderiza na interface

🧰 O que é um Framework?

"Um framework é uma estrutura de software reutilizável que fornece funcionalidades genéricas que podem ser customizadas para criar aplicações específicas."

Vantagens:

  • Acelera o desenvolvimento
  • Padronização de código
  • Segurança integrada
  • Comunidade e suporte

📚 Framework vs Biblioteca

Biblioteca:

⮕ Você chama a biblioteca quando precisa

⮕ Exemplo: lodash, axios


Framework:

⮕ O framework chama seu código (Inversão de Controle)

⮕ Exemplo: Express, Django, Spring


"Você usa uma biblioteca, mas um framework usa você."

🟢 Por que Node.js?

  • JavaScript Full-Stack: Mesma linguagem front e back
  • Assíncrono: Alta performance em I/O
  • NPM: Maior ecossistema de pacotes
  • Comunidade: Muito ativa e em crescimento
  • Empresas: Netflix, LinkedIn, Uber, PayPal
  • Modernidade: Tecnologia atual e relevante

🟢 O que é Node.js?

Node.js é um ambiente de execução JavaScript do lado do servidor, construído sobre o motor V8 do Chrome.


Características:

  • Runtime JavaScript fora do navegador
  • Event-driven e non-blocking I/O
  • Single-threaded com event loop
  • Ideal para aplicações em tempo real

🔁 Node.js Event Loop


// Código Síncrono (Bloqueante)
const data = fs.readFileSync('arquivo.txt');
console.log(data); // Espera terminar

// Código Assíncrono (Não-Bloqueante)
fs.readFile('arquivo.txt', (err, data) => {
  if (err) throw err;
  console.log(data);
});
console.log('Continua executando...');

🚀 Primeiro Servidor com Express


// Importar o Express
const express = require('express');
const app = express();

// Definir uma rota
app.get('/', (req, res) => {
  res.send('Olá, Mundo!');
});

// Iniciar o servidor
const PORT = 3000;
app.listen(PORT, () => {
  console.log(`Servidor rodando na porta ${PORT}`);
});

🧰 Servidor Node.js Sem Express


    // 1. Importar o módulo nativo do Node.js (não precisa de npm install)
    const http = require('http');

    // 2. Definir a porta
    const PORT = 3000;

    // 3. Criar o servidor
    // O callback (req, res) funciona de forma parecida, mas os objetos são mais simples
    const server = http.createServer((req, res) => {

      // No Express, o app.get('/') filtra a rota e o método automaticamente.
      // No Node puro, precisamos checar manualmente:
      if (req.url === '/' && req.method === 'GET') {

        // No Node puro, precisamos definir o status e o cabeçalho manualmente
        res.statusCode = 200;
        res.setHeader('Content-Type', 'text/plain; charset=utf-8');

        // O res.end() finaliza a conexão e envia o dado (equivalente ao res.send)
        res.end('Olá, Mundo!');

      } else {
        // Caso o usuário tente acessar outra rota (Ex: /sobre)
        res.statusCode = 404;
        res.end('Página não encontrada');
      }
    });

    // 4. Iniciar o servidor
    server.listen(PORT, () => {
      console.log(`Servidor rodando na porta ${PORT}`);
    });

🛣️ Rotas no Express


const express = require('express');
const app = express();

// Rota GET
app.get('/usuarios', (req, res) => {
  res.json({ mensagem: 'Lista de usuários' });
});

// Rota POST
app.post('/usuarios', (req, res) => {
  res.json({ mensagem: 'Usuário criado' });
});

// Rota com parâmetro
app.get('/usuarios/:id', (req, res) => {
  const id = req.params.id;
  res.json({ id: id, nome: 'João' });
});

🔗 Middlewares no Express

Funções que têm acesso aos objetos de requisição (req), resposta (res) e à próxima função middleware (next).


// Middleware de logging
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next(); // Passa para o próximo middleware
});

// Middleware para JSON
app.use(express.json());

// Middleware de autenticação
const auth = (req, res, next) => {
  if (req.headers.authorization) {
    next();
  } else {
    res.status(401).send('Não autorizado');
  }
};

🌐 API RESTful

REST: Representational State Transfer


Princípios:

  • Client-Server
  • Stateless (sem estado)
  • Cacheable
  • Interface uniforme
  • Sistema em camadas
