📋 O que são Listas?

🤔 Analogia do Catálogo Telefônico

📞 Catálogo Telefônico

  • 📖 Muitas páginas de contatos
  • 🔍 Você não lê todas as páginas
  • 👀 Só vê o que está na sua tela
  • ⏭️ Folheia conforme precisa
  • 🚀 Eficiente em memória

📱 Lista no App

  • 📋 Muitos itens de dados
  • 🔍 Não renderiza todos os itens
  • 👀 Só mostra os visíveis na tela
  • ⏭️ Carrega conforme o scroll
  • 🚀 Eficiente em performance

💡 Ideia Central: Assim como você não precisa ver todas as páginas de um catálogo ao mesmo tempo, uma lista no app não precisa renderizar todos os itens simultâneamente!

🧠 Por que Listas são Importantes?

  • 📊 Organização de Dados: Estruturam informações de forma clara
  • ⚡ Performance: Renderização eficiente de grandes volumes
  • 👆 Interação: Navegação intuitiva através de scroll
  • 📱 Limitação de Tela: Aproveitam melhor o espaço limitado
  • 🔄 Dinamismo: Podem ser atualizadas em tempo real

🎯 Exemplos Comuns:

Feed do Instagram • Lista de contatos • Produtos de e-commerce • Mensagens do WhatsApp • Notificações

🔧 Componentes de Lista no React Native

📋 FlatList vs ScrollView

📜 ScrollView

Analogia: Como imprimir uma lista inteira

  • 🖨️ Renderiza TODOS os itens
  • 📄 Como ter todas as páginas impressas
  • 💾 Usa muita memória
  • 🐌 Fica lento com muitos itens
  • ✅ Bom para poucos itens (<100)

📋 FlatList

Analogia: Como ler um livro digital

  • 👀 Renderiza só o que está visível
  • 📖 Como ver uma página de cada vez
  • 💨 Usa pouca memória
  • 🚀 Mantém performance estável
  • ✅ Ideal para muitos itens (1000+)

🎯 Regra de Ouro: Use ScrollView para listas pequenas e fixas. Use FlatList para listas dinâmicas e grandes!

📚 SectionList - Lista com Seções

📖 Analogia: Dicionário Organizado

Imagine um dicionário onde você tem:

  • 🅰️ Seções: Letras do alfabeto (A, B, C...)
  • 📝 Itens: Palavras dentro de cada letra
  • 🔍 Navegação: Fácil de encontrar palavras
  • 📑 Organização: Estrutura clara e lógica

📱 Exemplos de SectionList:

  • 📞 Lista de Contatos: Agrupados por letra inicial
  • 🛒 Carrinho de Compras: Produtos por categoria
  • 📅 Agenda: Eventos agrupados por data
  • 🎵 Playlist: Músicas por artista/álbum

🎭 Virtualização: A Mágica Por Trás das Listas

🎪 Analogia do Teatro

🎭 Teatro Tradicional

  • 👥 Todos os atores no palco
  • 💰 Caro manter todos pagos
  • 🏟️ Palco lotado e confuso
  • 📉 Performance ruim

🎪 Teatro Inteligente (Virtualizado)

  • 👀 Só atores na cena atual
  • 💸 Economiza recursos
  • 🎯 Foco na performance
  • 🔄 Atores entram/saem conforme necessário

🎯 Na Prática:

A FlatList mantém apenas ~10 itens renderizados (os visíveis + alguns extras), mesmo que sua lista tenha 10.000 itens!

⚙️ Como Funciona a Virtualização

🔄 Processo de Virtualização:

  1. 📏 Medição: O componente calcula o tamanho da tela
  2. 🧮 Cálculo: Determina quantos itens cabem na tela
  3. 🎨 Renderização: Cria apenas os itens visíveis + buffer
  4. 👆 Scroll: Usuário desliza a tela
  5. 🗑️ Reciclagem: Remove itens que saíram de vista
  6. 🆕 Criação: Adiciona novos itens que entraram em vista
  7. 🔁 Repetição: Processo continua infinitamente

💡 Resultado: Lista de 10.000 itens usa a mesma memória que uma de 10 itens!

🧭 Navegação entre Telas

🏠 Analogia da Casa

Imagine seu app como uma casa com vários cômodos:

  • 🚪 Telas = Cômodos: Cada tela é um ambiente diferente
  • 🚶 Navegação = Caminhar: Mover-se entre os cômodos
  • 📍 Estado = Localização: Saber onde você está
  • 🔙 Voltar = Memória: Lembrar de onde veio
  • 🗝️ Props = Objetos: Levar coisas entre cômodos

🎯 Exemplo Prático:

Sala (Lista de Produtos) → Quarto (Detalhes do Produto) → Banheiro (Carrinho de Compras)

📚 Tipos de Navegação

📚 Stack Navigation

Analogia: Pilha de livros

  • 📖 Coloca livro em cima
  • 🔙 Remove do topo para voltar
  • 📍 Sempre sabe qual é o atual

📑 Tab Navigation

Analogia: Fichário

  • 📋 Várias abas visíveis
  • 👆 Clica para alternar
  • ⚡ Acesso rápido

🍔 Drawer Navigation

Analogia: Menu lateral

  • 📜 Menu escondido lateralmente
  • 👈 Desliza para abrir
  • 📋 Lista de opções

📱 Modal

Analogia: Janela popup

  • 🪟 Aparece sobre a tela atual
  • ⚠️ Requer ação do usuário
  • ❌ Pode ser fechada

⚛️ React Navigation - O GPS do Seu App

🗺️ Analogia do GPS

🚗 GPS no Carro = React Navigation no App

  • 📍 Localização Atual: Sabe em qual tela você está
  • 🗺️ Mapa de Rotas: Define como as telas se conectam
  • 🧭 Navegação: Mostra como ir de A para B
  • 🔙 Voltar: Lembra o caminho de volta
  • 📱 Recalcular: Adapta-se a mudanças no percurso
  • 🏠 Tela Inicial: Seu ponto de partida (Home)

💡 Diferencial: Assim como o GPS te guia sem você precisar decorar todas as ruas, o React Navigation gerencia a navegação sem você se preocupar com os detalhes técnicos!

🎯 Conceitos Fundamentais

🏗️ Estrutura Básica:

  • 🌐 NavigationContainer: O "mapa" principal do app
  • 📚 Navigator: Define o tipo de navegação (Stack, Tab, etc.)
  • 📄 Screen: Uma tela individual do app
  • 🔗 Route: O "endereço" de cada tela
  • 🧳 Params: "Bagagem" que você leva entre telas

🎮 Métodos de Navegação:

  • 🚀 navigate(): "Vá para essa tela"
  • 📚 push(): "Adicione essa tela na pilha"
  • 🔙 goBack(): "Volte para a tela anterior"
  • 🏠 reset(): "Vá para o início e esqueça o histórico"

🧠 Gerenciamento de Estado

🧠 Analogia da Memória Humana

🧠 Estado = Memória do App

Assim como você precisa lembrar de coisas enquanto se move pela casa:

  • 📝 Memória de Curto Prazo: Estado local (useState)
  • 🗄️ Memória de Longo Prazo: Estado global (Context/Redux)
  • 📱 Anotações: Armazenamento persistente (AsyncStorage)
  • 🔄 Lembrar-se: Sincronizar estado entre telas

🎯 Exemplo: Você adiciona um item no carrinho (Tela A), vai para o produto (Tela B), volta para o carrinho (Tela A) e o item ainda está lá!

🏪 Tipos de Estado

🏠 Estado Local

Analogia: Gaveta do seu quarto

  • 📱 Só essa tela usa
  • 🔒 Privado da tela
  • ⚡ Rápido de acessar
  • 🗑️ Some quando tela é fechada

Exemplos: Estado do formulário, modal aberto/fechado, scroll position

🌐 Estado Global

Analogia: Geladeira da cozinha

  • 🏠 Toda a casa acessa
  • 👥 Compartilhado entre telas
  • 📊 Dados importantes do app
  • 💾 Persiste durante toda sessão

Exemplos: Dados do usuário, carrinho de compras, configurações, tema

🎁 Passando Dados Entre Telas

📦 Analogia do Correio

📮 Navigation Params = Envelope Postal

Imagine que você precisa enviar algo para alguém:

  • 📦 Pacote: Os dados que você quer enviar
  • 🏷️ Endereço: A tela de destino
  • 📮 Correios: O React Navigation
  • 📫 Caixa de Correio: Os params da tela
  • 📨 Entrega: A tela recebe os dados

💡 Vantagem: Assim como o correio, você pode enviar dados para telas que nem sabem quem está enviando!

🔄 Fluxo de Dados

📤 Enviando Dados:

Tela A → Tela B

navigation.navigate('TelaB', { usuario: 'João', idade: 25 })

📥 Recebendo Dados:

Na Tela B

const { usuario, idade } = route.params;

🔄 Cenários Comuns:

  • 🛒 Lista → Detalhes: ID do produto selecionado
  • 👤 Login → Home: Dados do usuário autenticado
  • ✏️ Formulário → Confirmação: Dados preenchidos
  • 📱 Lista → Editar: Item a ser editado
