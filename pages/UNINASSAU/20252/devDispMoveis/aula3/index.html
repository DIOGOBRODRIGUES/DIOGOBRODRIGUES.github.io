🎯 O que é JSX?

  • 🔤 JSX = JavaScript + XML: Sintaxe que parece HTML dentro do JavaScript
  • 🔄 Transpilação: Babel converte JSX para JavaScript puro
  • 📖 Legibilidade: Mais fácil de ler e escrever que JavaScript puro
  • ⚡ Produtividade: Desenvolvimento mais rápido e intuitivo

🔍 JSX - Regras Importantes

✅ Correto


// 1. className (não class)
<div className="container">

// 2. Fechamento obrigatório
<img src="foto.jpg" />
<input type="text" />

// 3. camelCase para atributos
<div onClick={handle} tabIndex={1}>

// 4. Expressões em {}
<p>{user.name}</p>
<h1>{2 + 2}</h1>

❌ Incorreto


// 1. class (palavra reservada JS)
<div class="container">

// 2. Tags não fechadas
<img src="foto.jpg">
<input type="text">

// 3. kebab-case (HTML tradicional)
<div onclick={handle} tab-index={1}>

// 4. Strings sem {}
<p>user.name</p>
<h1>2 + 2</h1>

🧩 Componentes Funcionais - Conceito Fundamental-📚 O que são Componentes?

  • 🏗️ Definição: Funções JavaScript que retornam JSX
  • 🧩 Modularização: Dividem a UI em peças independentes
  • ♻️ Reutilização: Podem ser usados várias vezes
  • 🔧 Manutenibilidade: Código organizado e fácil de debugar
  • 👥 Colaboração: Equipe pode trabalhar em componentes diferentes

🏗️ Anatomia de um Componente


// 1. Importações necessárias
import React from 'react';

// 2. Definição do componente (sempre com letra maiúscula!)
function Button({ title, onPress, disabled, type = 'primary' }) {
  // 3. Lógica do componente (se houver)
  const handleClick = () => {
    console.log(`Botão ${title} clicado!`);
    if (onPress) onPress(); // Chama função do pai se existir
  };

  // 4. Render (retorno do JSX)
  return (
    <button 
      onClick={handleClick} 
      disabled={disabled}
      className={`btn btn-${type} ${disabled ? 'btn-disabled' : ''}`}
      style={{
        padding: '10px 20px',
        borderRadius: '5px',
        border: 'none',
        cursor: disabled ? 'not-allowed' : 'pointer'
      }}
    >
      {title}
    </button>
  );
}

// 5. Exportação para uso em outros arquivos
export default Button;

📊 Props - Propriedades

🔍 Como funcionam

  • 📤 Pai → Filho: Dados fluem apenas para baixo
  • 🔒 Imutáveis: Componente filho não pode alterar
  • 📦 Objeto props: Recebidas como parâmetro

  • 🧪 Testabilidade: Entrada previsível = saída previsível
  • 🔧 Manutenção: Mudanças localizadas
  • ⚡ Performance: React otimiza com base nas props

🔥 useState - Estado no React -📚 O que é Estado (State)?

  • 💾 Definição: Dados que podem mudar ao longo do tempo
  • � Reatividade: Mudanças no estado atualizam a interface automaticamente
  • 🏠 Local: Cada componente pode ter seu próprio estado
  • ⚡ Performance: React otimiza re-renders baseado no estado

🎯 Comparação: Web Tradicional vs React useState

❌ Abordagem Tradicional


// HTML
<p id="contador">0</p>
<button id="btn">+1</button>

// JavaScript
let count = 0; // Variável global
document.getElementById('btn')
  .addEventListener('click', () => {
    count++; // Muda variável
    document.getElementById('contador')
      .textContent = count; // Atualiza DOM manualmente
  });

// ❌ Problemas:
// - Manipulação manual do DOM
// - Estado global (difícil controle)
// - Código repetitivo
// - Difícil de testar

✅ useState (React)


function Counter() {
  // Estado local do componente
  const [count, setCount] = useState(0);
  //     ↑       ↑              ↑
  //   valor   setter      valor inicial

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>
        +1
      </button>
    </div>
  );
}

// ✅ Vantagens:
// - React atualiza automaticamente
// - Estado encapsulado
// - Código limpo
// - Fácil de testar

📊 useState - Exemplo Completo


import { useState } from 'react';

function FormularioCompleto() {
  // Múltiplos estados no mesmo componente
  const [nome, setNome] = useState('');
  const [email, setEmail] = useState('');
  const [idade, setIdade] = useState(0);
  const [isLoading, setIsLoading] = useState(false);
  const [erro, setErro] = useState('');

  // Função assíncrona para enviar dados
  const handleSubmit = async (e) => {
    e.preventDefault(); // Previne reload da página
    setIsLoading(true); // Mostra carregamento
    setErro(''); // Limpa erros anteriores

    try {
      // Simulação de envio para API
      const response = await fetch('/api/usuarios', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ nome, email, idade })
      });

      if (!response.ok) throw new Error('Erro no servidor');
      
      // Sucesso - limpa o formulário
      setNome('');
      setEmail('');
      setIdade(0);
      alert('Usuário cadastrado com sucesso!');
      
    } catch (error) {
      setErro(error.message); // Mostra erro para usuário
    } finally {
      setIsLoading(false); // Remove carregamento
    }
  };

  return (
    <form onSubmit={handleSubmit}>
      <h2>Cadastro de Usuário</h2>
      
      {erro && <div className="erro">{erro}</div>}
      
      <input
        type="text"
        placeholder="Nome"
        value={nome}
        onChange={(e) => setNome(e.target.value)}
        required
      />
      
      <input
        type="email"
        placeholder="Email"
        value={email}
        onChange={(e) => setEmail(e.target.value)}
        required
      />
      
      <input
        type="number"
        placeholder="Idade"
        value={idade}
        onChange={(e) => setIdade(Number(e.target.value))}
        required
      />
      
      <button type="submit" disabled={isLoading}>
        {isLoading ? 'Enviando...' : 'Cadastrar'}
      </button>
    </form>
  );
}

🔄 Virtual DOM - O Segredo da Performance do React🤔 O que é o DOM Real?

  • 🌐 DOM (Document Object Model): Representação em árvore do HTML
  • 🐌 Lento: Manipulação direta é custosa computacionalmente
  • 🔄 Re-flows e Re-paints: Mudanças causam recálculos caros
  • ⚠️ Problemático: Muitas mudanças = interface travando

💻 Exemplo: Performance ruim com DOM tradicional


// ❌ DOM tradicional - MUITO LENTO para muitos elementos
for (let i = 0; i < 1000; i++) {
  const div = document.createElement('div');
  div.textContent = `Item ${i}`;
  document.body.appendChild(div); // 1000 manipulações do DOM!
}
// Resultado: Interface trava, usuário insatisfeito 😞

⚡ Virtual DOM - A Solução Inteligente

🔍 Como funciona

  • 💾 Cópia em memória: Representação virtual do DOM real
  • ⚡ Operações rápidas: JavaScript puro, sem acesso ao browser
  • 🔄 Diff Algorithm: Compara versões antigas vs novas
  • 🎯 Reconciliation: Atualiza apenas o que mudou

📊 Processo Step-by-Step

  1. Estado muda → Trigger re-render
  2. Nova árvore virtual → Criada em memória
  3. Diff → Compara antiga vs nova
  4. Patch → Aplica apenas diferenças
  5. DOM atualizado → Mínimo de mudanças

📈 Comparação de Performance: DOM vs Virtual DOM

❌ Manipulação Direta do DOM

  • 🐌 1000 mudanças: 1000 operações custosas
  • 🔄 Re-flows: Recálculo de layout a cada mudança
  • 🎨 Re-paints: Redesenho visual constante
  • ⏱️ Tempo: ~100ms+ para atualizar lista
  • 😞 UX: Interface trava e usuário frustra

✅ React Virtual DOM

  • ⚡ 1000 mudanças: ~10-50 operações reais no DOM
  • 🎯 Batch updates: Agrupa mudanças em lote
  • 🧠 Smart diffing: Algoritmo otimizado
  • ⏱️ Tempo: ~5-10ms para mesma lista
  • 😍 UX: Interface fluida e responsiva

🚀 Resultado: React é 10-20x mais rápido para atualizações complexas!

🔄 useEffect - Efeitos Colaterais -📚 O que são Efeitos Colaterais?

  • 🎯 Definição: Operações que afetam algo fora do componente
  • 🌐 Exemplos: APIs, DOM, timers, subscriptions, localStorage
  • ⏰ Quando executar: Após renderização, quando dependências mudam
  • 🔄 Lifecycle: Substitui componentDidMount, componentDidUpdate, componentWillUnmount

🎯 useEffect - Três Padrões Fundamentais

1️⃣ Executa Sempre


useEffect(() => {
  console.log('Toda renderização');
  // Executa a cada render
  // ⚠️ Cuidado: pode causar loops!
});

2️⃣ Executa Uma Vez


useEffect(() => {
  console.log('Apenas no mount');
  // Só executa quando componente monta
  // ✅ Ideal para: APIs, subscriptions
}, []); // Array vazio = sem dependências

3️⃣ Executa Quando Muda


useEffect(() => {
  console.log('userId mudou:', userId);
  // Executa quando userId muda
  // ✅ Ideal para: buscar dados específicos
}, [userId]); // Dependência específica
