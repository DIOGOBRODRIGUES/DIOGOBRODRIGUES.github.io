Lista de Requisitos para o Projeto (Vale 2 Pontos Extras)

1. Consumo da API

  • Implementação dos Métodos HTTP:
    • Utilizar adequadamente os métodos GET, POST, PUT e DELETE para interagir com a API.
    • As requisições devem ser realizadas de forma assíncrona, utilizando Promises ou async/await.
  • Tratamento de Respostas:
    • Implementar o tratamento correto das respostas da API, incluindo códigos de sucesso e erro.
    • Exibir mensagens claras ao usuário em caso de falhas nas requisições (ex.: erro de conexão, dados inválidos).

2. Integração Front-end e Back-end

  • Comunicação Eficiente:
    • Garantir que o front-end consuma corretamente os dados fornecidos pela API.
    • Atualizar a interface do usuário em tempo real após operações de criação, atualização ou exclusão.
  • Segurança:
    • Implementar medidas de segurança no consumo da API, como tokens de autenticação (se aplicável).

3. Usabilidade

  • Interface Intuitiva:
    • Desenvolver uma interface amigável e fácil de navegar.
    • Utilizar elementos visuais consistentes e padrões de design reconhecíveis.
  • Feedback ao Usuário:
    • Fornecer feedback imediato para as ações do usuário (ex.: loaders, confirmações, alertas).
    • Indicar claramente o sucesso ou falha das operações realizadas.

4. Componentização

  • Reutilização de Componentes:
    • Criar componentes React reutilizáveis para elementos recorrentes da interface.
    • Organizar o código de forma modular, facilitando a manutenção e escalabilidade.
  • Boas Práticas:
    • Seguir padrões de nomenclatura e estrutura de pastas para componentes, estilos e assets.

5. Funcionalidades Obrigatórias

  • Operações CRUD:
    • Implementar funcionalidades completas de Criação, Leitura, Atualização e Exclusão de dados.
  • Validação de Dados:
    • Realizar validações no front-end antes de enviar dados para a API.
    • Garantir a integridade e consistência das informações manipuladas.

6. Documentação

  • Comentários no Código:
    • Incluir comentários esclarecedores nas partes críticas do código.
  • Instruções de Uso:
    • Fornecer um guia ou README com instruções para execução do projeto e explicação das funcionalidades.

7. Apresentação do Projeto

  • Demonstração Prática:
    • Preparar uma apresentação demonstrando as principais funcionalidades implementadas.
    • Explicar como o front-end interage com a API e como os métodos HTTP são utilizados.
  • Slides de Apoio (Opcional):
    • Utilizar slides para destacar pontos-chave e facilitar o entendimento durante a apresentação.

10. Critérios Adicionais

  • Responsividade:
    • Assegurar que a aplicação seja responsiva e funcione bem em diferentes dispositivos e tamanhos de tela.
  • Acessibilidade:
    • Implementar práticas de acessibilidade para atender usuários com necessidades especiais.
  • Estilização e Layout:
    • Utilizar técnicas de CSS modernas (Flexbox, Grid) para um layout consistente.
    • Manter um design atraente e profissional.

Observações Finais

  • Prazos: Certifique-se de que o projeto seja entregue dentro do prazo estabelecido.
  • Originalidade: O trabalho deve ser original e desenvolvido pela equipe, evitando plágios.
