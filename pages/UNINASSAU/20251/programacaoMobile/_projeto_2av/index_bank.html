Serbank 2.0 - Requisitos da Segunda Entrega

Evolução do aplicativo bancário com funcionalidades avançadas

Visão Geral

Para a segunda entrega, o aplicativo Serbank deve evoluir incorporando funcionalidades mais avançadas, integrações com APIs externas e melhor experiência do usuário. O foco está em transformar o protótipo inicial em um produto mais completo e interativo, utilizando os conceitos avançados abordados nas aulas.

Objetivo: Aprimorar o aplicativo bancário Serbank com funcionalidades avançadas, integração com APIs externas e sensores do dispositivo, proporcionando uma experiência de usuário moderna e segura.

Requisitos Funcionais

1. Aprimoramento de Navegação

  • Stack Navigation: Implementar pilha de navegação para fluxos lineares (login → dashboard, detalhes de transação).
  • Tab Navigation: Menu inferior com acesso rápido às principais funcionalidades (Home, Transferências, Cartões, Perfil).
  • Drawer Navigation: Menu lateral para acesso a funcionalidades secundárias e configurações.
  • Deep Linking: Configurar rotas para acessar áreas específicas do app via links externos.

2. Dashboard Interativo

  • Visão Geral Financeira: Exibir saldo, receitas e despesas com gráficos interativos.
  • Cards de Acesso Rápido: Componentes para acessar funcionalidades principais.
  • Pull-to-Refresh: Atualização dos dados ao puxar a tela para baixo.
  • Notificações Personalizadas: Alertas sobre movimentações importantes na conta.

3. Listagem de Transações Otimizada

  • FlatList com Lazy Loading: Carregamento eficiente de grandes listas de transações.
  • Filtros e Ordenação: Opções para filtrar por data, valor e categoria.
  • Agrupamento: Transações agrupadas por data ou categoria.
  • Pesquisa: Busca por texto em descrições ou valores.
  • Detalhes de Transação: Tela detalhada ao clicar em um item da lista.

4. Integração com APIs

  • Cotações de Moedas: Exibir cotações atualizadas de principais moedas via API externa.
  • Simulador de Investimentos: Consumir API para cálculo de rendimentos.
  • Consulta de CEP: Autopreenchimento de endereços para cadastro e transferências.
  • Agendamento de Pagamentos: Sincronização com calendário do dispositivo.

5. Uso de Sensores

  • Autenticação Biométrica: Login e autorização de transações com TouchID/FaceID.
  • Detecção de Movimento: Bloqueio temporário ao detectar movimento incomum (segurança).
  • Modo Escuro Automático: Adaptação baseada no sensor de luminosidade.
  • Pagamento por Aproximação: Simulação utilizando NFC (onde disponível).

6. Transferências e Pagamentos Avançados

  • QR Code: Geração e leitura de QR Codes para pagamentos e transferências.
  • Transferências Agendadas: Programação de transferências futuras.
  • Pagamento de Boletos: Leitura de código de barras usando a câmera.
  • Contatos Favoritos: Lista de contatos frequentes para transferências rápidas.

7. Perfil e Configurações

  • Perfil Completo: Visualização e edição de dados cadastrais.
  • Preferências do App: Personalização da experiência do usuário.
  • Notificações: Gerenciamento de alertas e notificações.
  • Segurança: Configurações de segurança e privacidade.

Requisitos Não-Funcionais

1. Estilização e UI/UX Avançados

  • Tema Consistente: Implementar tema completo usando StyleSheet ou Styled Components.
  • Adaptabilidade: Layout responsivo para diferentes tamanhos de tela usando useWindowDimensions.
  • Animações: Transições suaves entre telas e componentes com Animated API.
  • Feedback Visual: Indicadores de carregamento, sucesso e erro.

2. Gerenciamento de Estado

  • Context API: Utilizar para gerenciar estado global da aplicação.
  • AsyncStorage: Persistência de dados do usuário e configurações.
  • Gerenciamento de Formulários: Validação e controle de estado em formulários.

3. Performance e Otimização

  • Memoização: Uso de useMemo e useCallback para otimizar renderizações.
  • Lazy Loading: Carregamento sob demanda de componentes e dados.
  • Prevenção de Memory Leaks: Limpeza apropriada de listeners e subscriptions.

4. Segurança

  • Validação de Inputs: Prevenção de injeção e validação de dados.
  • Timeout de Sessão: Logout automático após inatividade.
  • Mascaramento de Dados: Opção de ocultar saldo e informações sensíveis.
  • Simulação de Criptografia: Proteção de dados do usuário.

Escopo Mínimo (Requisitos Obrigatórios)

Para aprovação, o projeto deve implementar, no mínimo:

  1. Sistema de navegação completo (Tab + Stack) entre todas as telas.
  2. FlatList otimizada para exibição do histórico de transações.
  3. Integração com pelo menos uma API externa real.
  4. Uso de pelo menos um sensor do dispositivo.
  5. Persistência de dados com AsyncStorage.
  6. Estilização consistente e responsiva em todas as telas.
