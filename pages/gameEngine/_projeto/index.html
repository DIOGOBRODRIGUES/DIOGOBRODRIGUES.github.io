Requisitos do Projeto de Classe - Disciplina Game Engine

Objetivo

Desenvolver um protótipo funcional de um jogo simples utilizando a engine Godot (ou outra engine à escolha do aluno), aplicando os conceitos básicos vistos em aula, como câmera, física, interface de controle e saída, iluminação, textura, colisão e interação.

Requisitos Mínimos

  1. Câmera:
    • Implementar uma câmera funcional que acompanhe o jogador ou permita a visualização da cena de forma clara.
    • Exemplo: Câmera 2D ou 3D ajustada à movimentação do personagem ou cenário.
  2. Física:
    • Utilizar o sistema de física da engine para simular movimento ou gravidade.
    • Exemplo: Um personagem que cai ao pular ou um objeto que colide com o chão.
  3. Interface de Controle e Saída:
    • Criar controles básicos para o jogador (ex.: movimentação com teclado ou mouse).
    • Exibir uma saída simples na tela, como pontuação, vida ou mensagem de status.
  4. Iluminação:
    • Adicionar pelo menos uma fonte de luz ao cenário (ex.: luz ambiente, direcional ou pontual).
    • Garantir que a iluminação seja visível e funcional no jogo.
  5. Textura:
    • Aplicar texturas básicas a pelo menos um objeto ou personagem no jogo.
    • Exemplo: Textura em um chão, parede ou sprite do personagem.
  6. Colisão:
    • Implementar detecção de colisão entre objetos ou entre o jogador e o cenário.
    • Exemplo: O personagem não atravessa paredes ou interage com um obstáculo.
  7. Interação:
    • Criar pelo menos uma interação simples no jogo.
    • Exemplo: Coletar um item, abrir uma porta ou ativar um botão.

Especificações Gerais

  • O projeto pode ser 2D ou 3D, conforme a preferência do aluno.
  • O aluno pode escolher outra engine (ex.: Unity, Unreal Engine) desde que demonstre o cumprimento de todos os requisitos listados.
  • O jogo deve ser executável e entregue com instruções básicas de como rodá-lo.

Observações

  • Cada critério será avaliado com base na funcionalidade e na correta aplicação dos conceitos vistos em aula.
  • A "Apresentação Geral" considera a entrega no prazo, a jogabilidade básica e a ausência de erros críticos que impeçam o funcionamento.
  • Em caso de uso de outra engine, o aluno deve justificar brevemente a escolha e garantir que todos os requisitos sejam atendidos.
