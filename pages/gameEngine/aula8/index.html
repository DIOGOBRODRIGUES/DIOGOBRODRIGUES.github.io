Introdução ao 3D 🚀

Criar jogos 3D pode ser desafiador! A coordenada Z adiciona mais complexidade 🎯.

A boa notícia: o Godot usa APIs semelhantes para 2D e 3D! 🛠️

Sistema de Coordenadas 🗺️

📏 Godot usa sistema métrico em 3D: 1 unidade = 1 metro.

🎨 Sempre crie modelos na escala correta! Evita problemas em física e renderização.

🧭 Eixos:

  • X ➔ Lados
  • Y ➔ Cima/Baixo
  • Z ➔ Frente/Trás (direção principal)

🔵 Godot usa sistema destro para alinhar luzes, câmeras e objetos!

Comparação com outros softwares 3D 🧭

Manipulação de Objetos no 3D 🎮

Use manipuladores para mover, girar e dimensionar objetos!

🛠️ Eixos e cores:

🔴 X (vermelho) | 🟢 Y (verde) | 🔵 Z (azul)

Essas cores também valem para grades, shaders e vetores.

Conteúdo 3D no Godot

🎨 No desenvolvimento 3D, carregar conteúdo é mais complexo do que no 2D.

  • Necessário usar ferramentas especiais chamadas DCCs (Digital Content Creation).
  • O conteúdo precisa ser exportado para formatos de arquivo compatíveis (Exchange formats).
  • Ao contrário das imagens 2D, formatos 3D não são totalmente padronizados.

Geometria Gerada no Godot

🧩 É possível criar geometrias personalizadas programaticamente!

  • Utilize o recurso ArrayMesh para gerar formas customizadas.
  • Crie arrays de dados e use add_surface_from_arrays().
  • Ideal para modelos que não precisam ser atualizados constantemente.

SurfaceTool: Facilitando a Criação

🔧 Para facilitar a criação de geometrias, use a classe auxiliar SurfaceTool:

  • API mais intuitiva para criação de superfícies.
  • Ajuda na indexação, geração de normais e tangentes.
  • Permite criar malhas 3D de forma mais prática e rápida.

Atenção ao Desempenho

⚡ Criar vértices manualmente tem custo de desempenho!

  • Evite atualizar a geometria constantemente.
  • Use este método para criar geometrias estáticas no jogo.
  • Atualizações frequentes podem prejudicar o FPS.

Ambiente no 3D 🌎

🎨 Edite o ambiente usando o nó WorldEnvironment!

⚙️ Ajuste cor de fundo, adicione skybox e efeitos de pós-processamento.

🎥 O ambiente também pode ser sobrescrito pela câmera.

Visualização de Ambiente e Luz 💡

🔍 Sem WorldEnvironment ou DirectionalLight3D, o editor ativa luz/ambiente de pré-visualização.

🖥️ Essa luz só aparece no editor ➔ não é exportada!

🔧 Pode ser ligada/desligada nos ícones do menu superior.

Ajustes de Ambiente e Luz ⚙️

☑️ O menu de três pontos ao lado dos ícones permite configurar ambiente e luz de visualização!

🎛️ Use para ajustar propriedades se a pré-visualização estiver ativada.

Câmeras 🎥

👀 Sem uma Câmera 3D, nada será exibido na cena!

🎯 Câmeras podem usar projeção ortogonal ou perspectiva.

Comportamento das Câmeras 🎥✨

🎯 Câmeras são exibidas apenas na viewport pai ou avô.

🖥️ Cada subviewport precisa de sua própria câmera filha!

⚡ Regras principais:

  • 📸 Se nenhuma câmera, a primeira ativa automaticamente.
  • ✅ Se uma câmera tiver “atual” marcada ➔ ela é usada como prioridade.
  • 🔄 Se a câmera ativa sair, a próxima câmera na árvore assume.
